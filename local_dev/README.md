# EDINET 成長企業分析パイプライン

- `README.md`  
  成長率の定義、再現手順、考察、振り返りを記載しています。

- `data/output_example/figures/`  
  可視化の出力画像を保存しています。PNG形式で、成長企業ランキングや業種別の傾向を確認できます。

- `edinet_dbt/`  
  DuckDB上のデータをもとに、成長率計算や高成長企業テーブルの作成を行うdbtプロジェクトです。

- `main.py`, `scripts/`, `core/`  
  EDINET APIからのデータ取得、DuckDBへの読み込み、CSV出力、可視化作成を行うコードです。

- `Dockerfile`, `run_pipeline.sh`, `requirements.txt`, `.env.example`  
  Dockerまたはローカル環境でパイプラインを再現するために必要なファイルです。

なお、EDINETから取得した元データ、成⻑率40%以上の企業⼀覧などのデータはサイズが大きくなるため、提出物には含めていません。  
必要な場合は、以下の再現手順に従って、Dockerまたはローカル環境で再生成できます。
再生成時には、EDINET APIから `2026-01-01` から実行日当日までの期間を対象として、有価証券報告書データを取得することができます。

## Dockerで実行する場合（推奨）

Dockerを使用することで、Pythonやdbtの環境をローカルに直接構築せずに、同じ環境で一連の処理を実行できます。

### 1. Dockerイメージをビルド

    docker build -t edinet-growth .

### 2. Dockerコンテナを実行

    docker run --env-file .env -v ${PWD}/data:/app/data edinet-growth

このコマンドにより、以下の処理が順番に実行されます。

- EDINETメタデータの取得
- 有価証券報告書CSVの保存
- DuckDBへの元データ読み込み
- dbtモデルの実行
- dbtテストの実行
- 高成長企業一覧CSVの出力
- 可視化画像の作成

作成されたCSVと図は `data/output/` に保存されます。

## ローカル環境で実行する場合

Dockerを使わずに実行する場合は、以下の手順で実行できます。

### 1. 必要なライブラリをインストールする

    pip install -r requirements.txt

### 2. Pythonのパイプラインを実行する

EDINETメタデータの取得、有価証券報告書CSVの保存、DuckDBへの元データ読み込みを行います。

    python main.py

### 3. dbtモデルを実行する

`dbt run` では、DuckDB内の元データをもとに成長率計算や高成長企業テーブルを作成します。  
`dbt test` では、`schema.yml` に定義した `not_null` や `unique` などのテストを実行します。

    cd edinet_dbt
    dbt run --profiles-dir .
    dbt test --profiles-dir .
    cd ..

### 4. 可視化と出力CSVを作成する

dbtで作成した高成長企業テーブルをCSVとして出力し、可視化画像を作成します。

    python -m scripts.step4_visualization.export_dbt_outputs
    python -m scripts.step4_visualization.create_visualizations

作成されたCSVと図は `data/output/` に保存されます。

## 成長率の定義の選択理由

### 計算式（4年間CAGR）

    CAGR = (current_sales / sales_4_years_ago) ** (1 / 4) - 1

一般的に成長企業とは、複数年にわたって売上や純資産などが増加している会社を指します。  
本プロジェクトでは、純資産ではなく売上高にスコープを絞り、4年間の年平均成長率であるCAGRを成長率の指標として使用しました。

### 計算期間（2026/01/01〜現在）

有価証券報告書を使って分析する以上、企業の決算日と提出時期を考慮する必要があると考えました。日本企業は3月31日決算の会社が多く、有価証券報告書の提出には約3か月の猶予があります。今回作業した時点では5月17日であり、3月31日からまだ3か月経っていなかったため、最新の有価証券報告書だけを対象にすると、取得できる企業数が少なくなる可能性がありました。

そのため、最新のデータを使うか、1年前の期間（2025/01/01〜2025/12/31）を使って企業数を増やすかで迷いました。確認したところ、最新データでは約800件、1年前の期間では4000件以上の報告書が取得できました。

今回は、企業数よりも情報の新しさを優先し、最新データを使う方を選びました。ただし、その分だけ対象企業数は少なくなるため、分析の網羅性には制約があります。そのため、今回の結果は全上場企業全体の傾向ではなく、取得期間内に対象書類を提出し、売上高データを取得できた企業に対する分析として扱いました。


## 可視化と考察

### 4年間CAGRに基づく高成長企業ランキング

![chart](data/output_example/figures/top_four_year_cagr.png)

4年間CAGRでランキングを作成したところ、株式会社ＲＥＶＯＬＵＴＩＯＮが約137.3%で最も高い結果となりました。4期前の売上高は約10.9億円でしたが、当期売上高は約345.7億円まで増加しており、売上規模が大きく変化していることが分かりました。

2位の株式会社メタプラネットもCAGRが約103.6%となっており、平均すると毎年売上が約2倍のペースで成長している計算になります。このようにCAGRを見ることで、一定期間における企業の成長スピードを分かりやすく比較できました。

一方で、CAGRは初期の売上高が小さい企業ほど高く出やすいという特徴があります。そのため、CAGRのランキングだけを見て企業の成長力や事業の安定性を判断するのは危険だと感じました。

より正確に成長の実態を理解するためには、CAGRだけでなく、当期売上高や売上増加額も合わせて確認する必要があると考えました。これにより、CAGRの高い企業が実際に大きな売上規模まで成長しているのか、それとも小さい売上から一時的に大きく伸びたのかを、より深く考察できると思います。


### 売上規模と売上成長率の関係

![chart](data/output_example/figures/growth_vs_sales_size.png)

このグラフでは、売上規模と4年間CAGRの関係を確認しました。本分析では、売上高に基づいて企業規模を複数のグループに分類しました。今回のCAGR40%以上の企業では、主に「10億円未満」「10億円〜100億円」「100億円〜1000億円」の3つのグループに分布しました。

その結果、CAGR40%以上の企業では、売上10億円〜100億円のグループが7社で最も多くなりました。ただし、売上10億円未満のグループとの差は1社だけであるため、売上10億円〜100億円の企業が最も成長しやすいとは言い切れません。

一方で、売上100億円〜1000億円のグループは他のグループより会社数が少なかったため、今回のデータでは、売上規模が大きくなるほど高いCAGRは出にくい可能性があると感じました。ただし、対象データ数が限られているため、このグラフだけで強く結論づけるのは難しいと考えています。


### 高成長企業の業種構成

![chart](data/output_example/figures/high_growth_industry_pie.png)

このグラフでは、高成長企業の業種構成を確認しました。結果として、情報・通信業とサービス業がそれぞれ25.0%で最も多く、次に卸売業と不動産業が18.8%、機械と小売業が6.2%となりました。

今回のデータでは、情報・通信業とサービス業が高成長企業の中で多く見られました。ただし、対象データ数が限られていることや、業種によって決算期が異なる可能性があるため、この結果だけで特定の業種が高成長になりやすいとは言い切れないと考えています。今後は、取得期間を広げたり、業種をさらに細かく分類したりすることで、より深い分析ができると感じました。


### 業種別平均4年CAGRランキング

![chart](data/output_example/figures/industry_avg_cagr.png)

業種構成の円グラフでは、情報・通信業とサービス業の企業数が最も多い結果となっていましたが、平均CAGRで見ると、不動産業が84.7%で最も高い結果となりました。つまり、高成長企業の「数」が多い業種と、「平均成長率」が高い業種は必ずしも一致しないことが分かりました。

ただし、今回のデータ数は限られており、平均値は一部の大きく成長した企業の影響を受けやすい傾向があります。そのため、この結果だけで不動産業が最も成長しやすいとは言い切れません。今後は業種ごとの企業数に加えて、中央値や標準偏差も確認し、平均CAGRの高さが一部の企業によるものなのか、業種全体の傾向なのかを見分ける必要があると感じました。


## 結論

今回の分析では、EDINETから取得した有価証券報告書のデータをもとに、売上規模、業種、4年間CAGRの関係を確認しました。結果として、CAGR40%以上の高成長企業は、売上10億円〜100億円のグループにやや多く見られました。ただし、売上10億円未満のグループとの差は小さく、売上規模だけで成長しやすさを判断するのは難しいと感じました。

一方で、売上100億円〜1000億円のグループでは、CAGR40%以上の企業数が他のグループより少ない結果となりました。このことから、今回取得したデータ内では、売上規模が大きくなるほど高い成長率を維持するのは難しくなる可能性があると考えました。

業種別に見ると、高成長企業の数では情報・通信業とサービス業が多く見られましたが、平均CAGRでは不動産業が最も高く、企業数が多い業種と平均成長率が高い業種は必ずしも一致しないことが分かりました。この点から、高成長企業を分析する際には、単純に企業数だけを見るのではなく、平均成長率、個別企業の売上規模、売上増加額も合わせて確認する必要があると感じました。

ただし、今回の分析は取得期間や対象企業数に限りがあり、また業種によって決算期や売上構造も異なります。そのため、この結果だけで「どの業種・どの売上規模が成長しやすい」と強く結論づけることは難しいと考えています。今後は取得期間を広げたり、中央値や標準偏差も確認したりすることで、一部の大きく成長した企業の影響なのか、業種全体の傾向なのかをより正確に分析できると感じました。

## dbtモデル構成の意図

dbtのモデルは、処理の役割ごとに `staging`、`intermediate`、`marts` の3つに分けました。

- `staging` では、EDINETのメタデータから有価証券報告書と訂正有価証券報告書だけを抽出し、会計期間の欠損がある場合は `docDescription` から引用しました。

- `intermediate` では、同じ企業・同じ会計期間の書類が複数ある場合に最新の提出書類だけを残し、企業ごとの最新の有価証券報告書を選びました。その後、CSVから売上高を抽出し、成長率やCAGRを計算しました。

- `marts` では、最終的な分析や可視化で使うテーブルを作成しました。成長率の情報に加えて、EDINETコードリストから取得した業種などを結合し、業種別の考察ができるようにしました。

## パイプライン全体を複数回実行した場合の挙動

- 有価証券報告書CSVは、同じ `docID` のファイルがすでに存在する場合、再ダウンロードせずにスキップします。
- DuckDBのテーブルは、再実行時に `CREATE OR REPLACE TABLE` で作り直されます。
- dbtモデルでも、`dbt run` を再実行するとモデル定義に基づいて再作成されます。そのため、同じデータが重複して追加され続けることはありません。
- ただし、EDINET側に新しい提出書類や訂正報告書が追加された場合は、再実行したタイミングによって結果が変わる可能性があります。
